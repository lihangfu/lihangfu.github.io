接口幂等性

保证接口的幂等性，防止一个接口在特殊情况下被重复请求时依旧保持结果的有效，不会重复消费接口

先说一下使用 redis 实现接口幂等性的思路：

  1. 在访问需要保证接口幂等性的接口前单独请求一次 token
  2. 后端将该 token 放入redis，并设置有效时间
  3. 前端带着该 token 发起请求
  4. 后端验证该 token 是否存在，存在则删除，且删除成功后，正常执行接口任务
  5. 若不存在或是删除失败，则已经被消费，返回重复消费的提示
  • 首先定义 RedisService ，提供三个方法
  • 提供操作 token 的 TokenService
  • 定义注解，通过注解动态绑定需要接口幂等性的接口
  • 全局异常处理器
  • 自定义异常
