介绍

三大主流框架：

  • jackson
  • gson
  • fastjson

在 SpringMVC 框架中， jackson 和 gson 已经自动配置好，添加依赖即可使用，fastjson 需要自己配置。

序列化和反序列化

  • 序列化：对象 -> JSON （响应处理）
  • 反序列化：JSON -> 对象（请求处理）

HttpMessageConverter

转换器：对象 -> JSON，JSON -> 对象

HttpMessageConverter 是 Spring 定义得一个转换接口，在 JSON 的流行框架中都进行了各自的实现。

  • jcakson：MappingJackson2HttpMessageConverter
  • gson：GsonHttpMessageConverter
  • fastjson：FastJsonHttpMessageConverter（需要添加 fastjson 依赖）

SpringMVC 中 jackson 和 gson 的 HttpMessageConverter 已经自动配置好，添加依赖即可使用，fastjson 需要开发者手动配置 HttpMessageConverter。

异常页面定义

在前后端不分离项目中，如何在出现异常时，自动跳转到异常页面

在 SpringBoot 中，已经做好了一切的配置，只需要在 static 或者 templates 目录下新建 error 目录提供异常页面即可。

页面命名规范有两种：

  1. 具体状态码：404.html
  2. 范围状态码：5xx.html

也可以使用动态模板展示，命名规范相同，优先级：具体状态码 > 范围状态码 >动态 > 静态 > 范围状态码

原理

原理配置在 ErrorMvcAutoConfiguration类中的conventionErrorViewResolver方法向容器中注入默认的异常处理类 DefaultErrorViewResolver

DefaultErrorViewResolver类实现了ErrorViewResolver接口的主要方法resolveErrorView

该方法先是获取当前异常状态码，然后在指定位置查找动态模板，差不多查静态模板，查不到返回 null

拿到 null 后去 map 中找对应的 4xx 或 5xx 然后再去查动态，查静态并返回结果

跨域

SpringBoot 提供了三种跨域方案

基础概念

  • CORS（Cross-Origin Resource Sharing）：跨域资源共享

  • 域：站点，由协议、域名（ IP ）、端口三部分组成，三者有一种不同则属于不同域

  • 资源：url对应的资源，可以是图片、HTML、JSON数据都属于资源
  • 同源策略：主要为防止 XSS 攻击，浏览器客户端应该仅请求与当前页面的来自同一个域的资源
  • 预检请求：当请求不是一个简单的请求时，会发起一个预检请求，比如 PUT、DELETE 请求，相当于一个嗅探的功能，查看服务端是否支持我接下来要发送的请求，服务端会告诉预检请求服务端都支持哪些请求，你可以放心发生，客户端收到之后才会发生真正的请求。
