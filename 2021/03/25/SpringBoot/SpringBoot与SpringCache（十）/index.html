Spring Cache

Java 缓存体系多且复杂，为了降低学习成本，使用 Spring Cache

Spring Cache 是缓存体系的抽象实现，类似与 JDBC 的存在

只提供一个规范，需要提供具体实现：例如 Ehcache、Redis等

下面案例以 Redis 为例

@EnableCaching

开启 SpringCache ：配置类上添加 @EnableCaching 注解开启

  1. proxyTargetClass：默认 false ，使用基于 jdk 的动态代理，为 true 时使用基于 CJLB 的动态代理

    缓存最终作用与业务层，使用代理对象代理业务层进行缓存操作

新建缓存

@Cacheable

标记在方法上，表示该方法的返回结果需要缓存

方法执行前，根据规则去获取缓存结果，获取不到则执行原方法，并对返回值进行缓存

通常作用于业务层，对该方法进行缓存，代理对象直接从缓存服务器获取该返回值返回

  1. cacheNames：缓存 key 的前缀

  2. key：缓存的 key ，默认情况下以方法的参数为 key，多个参数则多个参数共同作为缓存的 key

    自定义 key： @Cacheable(cacheNames = “user”,key = “#id”) // 参数 id 为 key

    ​ @Cacheable(cacheNames = “user”,key = “#root.method.name”) // 方法名为 key

  3. keyGenerator：自定义 key 生成器

    设置该 key 生成器生效：@Cacheable(cacheNames = “user”,keyGenerator = “myKeyGenerator”)

缓存更新

当我们去修改一个已经被缓存了对象时，我们再次调用缓存方法会发现，缓存对象并没有被修改。

此时我们需要进行缓存更新，让缓存的数据和我们修改的数据进行同步

@CachePut：缓存不存在则进行缓存，存在则更新

  1. cacheNames：缓存 key 的前缀
  2. key：需要更新的数据的 key，具体设置方法可以参照新建缓存时 key 的设置

缓存清除

如果我们对一个已经被缓存的对象执行删除操作，但缓存没更新，则导致我们依然可以拿到该对象

@CacheEvict

  1. cacheNames：缓存 key 的前缀
  2. key：需要删除的数据的 key，具体设置方法可以参照新建缓存时 key 的设置
  3. condition：满足条件才进行删除（不常用）
  4. allEntries：默认 false，是否删除当前 cacheNames 下所有的缓存
  5. beforeInvocation：默认 false，先执行方法再删除，true 时先删除在执行方法

补充

  • @Caching：可以组合使用 @Cacheable、@CachePut、@CacheEvict
  • @CacheConfig：在类上配置该类所有方法缓存的 cacheNames 和 keyGenerator

配置含义：

  1. name:缓存名称，可以通过 cache 注解的 cacheNames 属性选择对应规则的配置。
  2. maxElementsInMemory：缓存最大个数。
  3. eternal:对象是否永久有效，一但设置了，timeout将不起作用。
  4. timeToIdleSeconds：设置对象在失效前的允许闲置时间（单位：秒）。仅当eternal=false对象不是永久有效时使用，可选属性，默认值是0，也就是可闲置时间无穷大。
  5. timeToLiveSeconds：设置对象在失效前允许存活时间（单位：秒）。最大时间介于创建时间和失效时间之间。仅当eternal=false对象不是永久有效时使用，默认是0.，也就是对象存活时间无穷大。
  6. overflowToDisk：当内存中对象数量达到maxElementsInMemory时，Ehcache将会对象写到磁盘中。
  7. diskSpoolBufferSizeMB：这个参数设置DiskStore（磁盘缓存）的缓存区大小。默认是30MB。每个Cache都应该有自己的一个缓冲区。
  8. maxElementsOnDisk：硬盘最大缓存个数。
  9. diskPersistent：是否缓存虚拟机重启期数据。
  10. diskExpiryThreadIntervalSeconds：磁盘失效线程运行时间间隔，默认是120秒。
  11. memoryStoreEvictionPolicy：当达到maxElementsInMemory限制时，Ehcache将会根据指定的策略去清理内存。默认策略是LRU（最近最少使用）。你可以设置为FIFO（先进先出）或是LFU（较少使用）。
  12. clearOnFlush：内存数量最大时是否清除。
  13. diskStore 则表示临时缓存的硬盘目录。
