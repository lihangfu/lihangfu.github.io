SpringBootDevTools与单元测试（九）

发表于2021-03-25|更新于2021-03-25|SpringBoot
|阅读量:

DevTools

监控 classpath(target) 目录下文件变化，自动重启项目。

静态资源的修改并不会引发重启！

防止模块间传递，不安全！

注意：

​ 在 eclipse 中保存后触发自动编译，项目自动重启。而 IDEA 自动保存，并不会即时编译。

​ 所以 IDEA 需要点击编译按钮才会自动重启，需要进行配置才可以完全自动重启项目。

IDEA 配置

  1. File -> Settings -> Build,Execution,Deployment -> Compiler
  2. 勾选 Build project automatically

  1. CTRL + SHIFT + ALT + / 选择 Registry
  2. 勾选 compiler.automake.allow.when.app.running

  1. 配置完成后，运行代码，修改代码后一定时间后就会自动重启（不推荐使用）

静态资源监控重启（不推荐）

设置里开启自动 build 后，修改静态文件就会自动编译，但静态文件的访问无需重启加载

静态资源不需要重启就可以直接访问修改后的内容

所以不推荐监控静态资源进行重启！！！

  • 开启静态资源监控重启
  • 监测变化但不重启的静态资源目录（只触发 LiveReload）：
    • /META-INF/maven
    • /META-INF/resources
    • /resources
    • /static
    • /public
    • /templates

禁用自动重启

控制重启行为

  • 在 resources 目录下创建 .reloadtrigger 文件（文件名随意，在上诉配置中指定即可）

  • 此时自动重启必须满足以下条件

    • 修改代码
    • 修改 .reloadtrigger 文件

全局配置

配置全局配置后，项目添加 devtools 后就会自动应用配置

  1. 在当前用户目录下新建 .spring-boot-devtools.properties 文件
  2. 在该文件中添加全局配置（所有使用 devtools 的项目都会自动应用配置）

单元测试

Spirng Boot 添加依赖时会自动添加对应的单元测试依赖

  • Spring Boot 自动创建的测试模板

@SpringBootTest 参数详解

  • webEnvironment：指定 web 应用环境
    • MOCK：提供模拟的 servlet 环境
    • RANDOM_PORT：提供了一个真正的 servlet 环境，以随机端口启动
    • DEFINED_PORT：提供了一个真正的 servlet 环境，以默认端口启动
    • NONE：不提供 web 环境
  • classes：指定应用的启动类（无需设置，可以自动扫描到启动类）

单元测试回滚

在单元测试操作数据库时，需要添加 @Transactional 注解，测试完成后会自动回滚

注意：

​ 设置 webEnvironment 为真实 servlet 环境时，由于设计多线程，并不会自动回滚！！！
